Detailed Description

ICARUS OS Task Management API.

Version
0.1.0

Public API for task creation, scheduling, and lifecycle management. This is the primary interface for application code to interact with the ICARUS kernel.

Example Usage:
// Register a task with the kernel
os_register_task(my_task_function, "MyTask");
// Start the scheduler
os_start();
// Inside a task: yield CPU voluntarily
os_yield();
// Inside a task: sleep for 100 ticks
task_active_sleep(100);

void os_init ( void )

Initialize the ICARUS kernel.

Must be called before any other kernel function. Initializes:

  • Task pool and task list
  • Semaphore pool
  • Message pipe pool
  • System tasks (idle, heartbeat)
Precondition

Postcondition
Kernel is ready for task registration
Note
This function registers two system tasks:
  • ICARUS_KEEPALIVE_TASK (idle task)
  • ICARUS_HEARTBEAT_TASK (LED heartbeat)

◆ os_yield()

void os_yield ( void )

Voluntarily yield CPU to scheduler.

Triggers a context switch to the next ready task. The current task is placed in READY state and will be scheduled again.

Note
This function is placed in ITCM for zero wait-state execution.
Implementation:
Sets PendSV pending bit to trigger context switch at lowest interrupt priority, ensuring clean stack state.
See also
ARMv7-M Architecture Reference Manual, Section B1.5.14 (PendSV)

Sleep for specified number of ticks (non-blocking)

Blocks the current task for the specified duration while allowing other tasks to run. This is the preferred sleep method.

Parameters

Returns
Actual number of ticks elapsed (may be >= requested due to scheduling)
Note

Example:
